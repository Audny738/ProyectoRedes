Seguridad en la web



Amenazas a la Seguridad Web

La definición de amenaza consiste en un evento que puede dañar y perjudicar un sistema de información. Ahora, una amenaza a la seguridad es un riesgo que puede dañar sistemas computacionales y organizaciones. La principal intención de un ataque a la seguridad robar, alterar o destruir una pieza de información personal y confidencial, robar el espacio del disco duro y acceder ilegalmente a las contraseñas.

Por lo general, la Seguridad Web lidia con dos tipos de atacantes: los atacantes web y los atacantes de redes Un atacante web controla al menos un servidor que responde a cualquier solicitud enviada con contenido malicioso. Los atacantes de red extienden las capacidades de los atacantes web con la habilidad de detectar e interceptar todo el tráfico enviado entre dos endpoints.

Debido a la gran cantidad de ataques que existen en la web, es importante que los desarrolladores cuenten con un lugar que cuente con toda esta información. Por suerte, este lugar existe conocido como OWASP.


OWASP


El Proyecto de Aplicaciones Web Abiertas (OWASP) es una fundación sin fin de lucro que trabaja para mejorar la seguridad del software. Cuenta con una gran comunidad que lidera una gran de cantidad de proyectos Open-Source, cientos de capítulos con información, miles de miembros yconferencias. De esta manera se convierte en la referencia principal de los desarrolladores para asegurar la web.

Esta fundación también cuenta con el proyecto OWASP Top 10. En este documento se listan las 10 vulnerabilidades más comunes de las aplicaciones. También muestra los riesgos, impactos y las contramedidas.

IMPORTANTE: El Top 10 de riesgos es actualizado cada 3-4 años en donde reevalua a todas las nuevas amenazas junto con las antiguas para determinar cuál representa mayor peligro. La última actualización que se tuvo es tl Top Ten 2021, el cual se basa la información que se redacta en esta página. Para una consultar las versiones futuras puede consultarlas aquí.

OWASP Top 10 Amenazas

  1. Inyecciones: Suceden cuando un atacante envía información inválida a la aplicación web con la intención de que esta haga una acción para la cual no estuvo diseñada/programada.
  2. Autenticación Rota: Puede hacer que un atacante utilice métodos manuales o automáticos para obtener el control sobre cualquier cuenta que ellos quieran en un sistema o inclusive, sobre todo el control del sistema. Existen varios tipos.
  3. Exposición de datos sensibles: Consiste en comprometer la información que debería ser protegida. Algunos datos sensibles pueden ser: credenciales, información médica, número de tarjeta de crédito, etc.
  4. Entidades Externas XML (XXE): Es un tipo de ataque contra una aplicación que parsea la entrada XML. El ataque ocurre cuando la entrada XML contiene una referencia a una entidad externa.
  5. Control de Acceso Roto:Es cuando no se limita el acceso a los visitantes sobre que páginas o secciones pueden navegar generando que ellos mismos puedan: ver archivos sensibles, acceder a funcionalidades, cambiar los derechos de acceso.
  6. Fallas en la configuración de la Seguridad: Existen varias variantes de este ataque como: archivos y directorios sin protección, servicios innecesarios, páginas sin uso, usar las configuraciones predeterminadas, etc.
  7. Cross-Site Scripting (XSS): Consiste en inyectar scripts maliciosos del lado del cliente usando el sitio web como método de propragación.
  8. Deserialización insegura:El proceso de deserialización consiste en convertir cadenas de bytes a objetos que pueden ser perjudiciales para el sitio.
  9. Usar componentes con vulnerabilidades conocidas: Si no actualizamos el software tanto en el Frontend como en el Backend del sitio se corre un alto grado de riesgo de introducir atacantes a nuestro sitio.
  10. Registro y supervisión insuficientes: El hecho de tener deficiencias en el proceso de monitoreo y registro aumenta las probabilidades de daños en el sitio, generando vulnerabilidades en aplicacciones como: el DNS, el lado del servidor de la aplicación, etc.
